# Code audit — 2026-06-21

A deep architectural + implementation audit of the multi-agent-sdlc-crew
profile, scoped to the team-adoption goal: a shared, mandatory SDLC path that
every team member's code passes through identically. Findings are concrete and
prioritized. The coverage mandate scoped to this audit is now **complete**;
this document records the baseline, the architectural findings, the
remediation that landed, and the one deliberately-left item.

## Scope and measurement

- Shell core: `claudecfg/hooks/lib.sh` (1576 lines, 74 functions), the per-hook
  scripts, `scripts/validate.mjs` (673 lines), and the benchmark scripts.
- Python: `scripts/*.py` (3759 lines). Measured with branch coverage via
  `make cov` (added in this audit). Baseline: **45.29%** branch coverage on
  `scripts/*.py`, 255 tests passing, with a ratcheting `--cov-fail-under` gate
  at 45 so coverage could only rise.
- Final state: **100%** line + branch coverage on `scripts/*.py` (1924 stmts /
  0 missed / 824 branches / 0 missed), **648 tests** passing. The `COV_MIN`
  gate is ratcheted to 100. The gate applies to `make cov` only, **not** to
  `make test`, so existing CI is unaffected.
- Bash has no line-coverage tooling in this audit; behavioral coverage is via
  `tests/hooks/cases.json` (140 cases) + `tests/hooks/scenarios.json` (2
  stateful scenarios) + `scripts/test-hooks.sh`, plus direct function unit
  tests in `tests/hooks/test-lib.sh` (~273 direct assertions over ~49 `lib.sh`
  enforcement functions) and the Python suites
  (`tests/bench/test_hook_effective_roles.py`, `test_message_mentions.py`,
  `test_concrete_outcome_recognition.py`).

## Architectural findings

1. **lib.sh is a well-decomposed monolith.** 74 small, focused functions is
   good decomposition, but they all live in one 1576-line sourced file mixing
   five concerns: state + locking, transcript parsing, subagent
   canonicalization, command classification, and footer-contract enforcement.
   Splitting a sourced lib is risky (sourcing order, shared globals), so this
   is low priority — the decomposition is the real quality signal and it is
   sound. Noted, not acted on.

2. **The state lock is correct and portable.** `_acquire_state_lock` uses
   `mkdir` (atomic on every Unix since 4.3 BSD) instead of Linux-only `flock`,
   with stale-lock recovery keyed on lock age (not PID liveness, avoiding
   PID-reuse hangs), a missing-epoch streak counter, and a
   consecutive-failed-break backstop so the agent can never block indefinitely.
   This is the right design for concurrent team/multi-agent use. No change
   needed.

3. **The footer contract is the enforcement heart.** `message_mentions_*` +
   `block_checklist_*` + `session_*_reason` decide whether a stop/subagent-stop
   is blocked. It is behavior-rich (many accepted phrasings, bilingual EN/RU).
   This is where "mandatory identical path" actually lives for the team. It is
   the highest-value place to invest tests.

4. **No new skills/commands/agents can be added** — `scripts/validate.mjs`
   enforces fixed inventories (5 skills, 9 commands, fixed agent frontmatter).
   This is intentional for profile stability and is correct for team use: the
   workflow cannot silently drift. SDD therefore ships as a workflow doc, not
   a skill.

## Implementation findings (from the baseline)

1. **49 of 74 lib.sh functions had no direct unit test** (only indirect
   coverage via cases.json/scenarios, if any). Highest-value gaps for the
   enforcement core: the `detect_*_cmd` family, command classification
   (`is_dangerous_rm_command`, `is_force_push_command`,
   `is_remote_shell_bootstrap_command`, `command_is_hard_denied_by_profile`),
   the `block_checklist_*` builders, `session_block_reason`, and the lock
   internals. These decide the mandatory path and were the highest-value place
   to lock with direct tests.

2. **Two python files were at 0% coverage**: `scripts/bench_runner_openrouter.py`
   (141 lines) and `scripts/find-failed-benchmark-run.py` (84 lines). No tests
   existed at all.

3. **`scripts/bench_runner_claude_code.py` (2208 lines) was 51%** — the entire
   `main`/CLI block (lines ~1725-2204) was untested. This was the largest
   single file and the largest risk surface.

4. **Other low-coverage scripts**: `wait-for-benchmark-slot.py` (23%),
   `select-benchmark-tasks.py` (46%), `download-benchmark-summary.py` (52%),
   `build-benchmark-matrix.py` (68%).

5. **Duplication in the footer-contract family** (remediated this audit): the
   seven `message_mentions_*` functions each repeated a `message_has_line_prefix
   "$msg" "X:" || ...` chain. Extracted `message_has_any_line_prefix` helper;
   behavior identical (verified: 648 tests, 140 cases, 2 scenarios, lint,
   shellcheck, validate all green). Net reduction ~90 lines of repetition.

## Remediation done

### Refactors

- `claudecfg/hooks/lib.sh`: extracted the `message_has_any_line_prefix` helper
  and deduplicated the seven `message_mentions_*` functions
  (behavior-identical, ~90 lines removed).
- `scripts/bench_runner_claude_code.py`: extracted the pure
  `classify_task_failures(...)` function from `main()` so failure
  classification is unit-testable without a live runner; pre-initialized the
  `completed` flag before the 429-retry loop in `run_claude` (fixes a latent
  `NameError` if `OLLAMA_429_MAX_RETRIES=0`).
- `scripts/find-failed-benchmark-run.py`: moved `from datetime import
  timedelta` to the top-level import (out of the function body).

### Bugfixes

- `scripts/compare-benchmarks.sh`: the `deltas:` jq object was missing
  `clean_pass_rate`, `recovered_task_rate`, and `summary_repair_rate`, which
  `render-benchmark-report.sh` reads. The report crashed with jq exit 5 on
  real compare output. The three deltas are now emitted.
- `scripts/bench_runner_claude_code.py`: `forbidden_doc_pattern_hits` was fed
  a snapshot dict instead of its `.text` string, raising a `TypeError`. Now
  fed the string.

### Coverage infrastructure

- `pyproject.toml` coverage config: `branch = true`, `source = ["scripts"]`,
  `relative_files = true`, and `exclude_lines` for `if __name__ == "__main__"`,
  `NotImplementedError`, and `pragma: no cover`.
- `Makefile`: `COV_MIN` ratcheted 45 → 100; the `cov` target runs
  `pytest --cov=scripts --cov-branch --cov-fail-under=$(COV_MIN)`. The gate
  applies to `make cov` only, not `make test`.
- `make hooks` now runs `bash tests/hooks/test-lib.sh`; `make lint` adds
  `tests/hooks/test-lib.sh` to the `bash -n` + shellcheck targets;
  `scripts/validate.mjs` adds `tests/hooks` to its bash `-n` find and the
  shellcheck targets; the Hook Contracts CI workflow
  (`.github/workflows/hooks-test.yml`) gained a `Run lib.sh unit tests` step.

## Outcome / status

The coverage mandate is **complete**. Final, verified-green state:

- `make lint` → "All checks passed!"
- `python3 -m pytest tests/ -q` → 648 passed
- `make cov` → TOTAL 1924 stmts / 0 missed / 824 branches / 0 missed / 100%
  (gate `COV_MIN=100`, applies to `make cov` only, not `make test`)
- `make hooks` → green, including `tests/hooks/test-lib.sh` (273 direct
  assertions over ~49 `lib.sh` enforcement functions)
- `node scripts/validate.mjs` → all checks passed

Baseline at the start of this audit: 45.29% branch coverage on `scripts/*.py`,
255 tests. Now 100% / 648 tests.

### Deliberately left

- **No measured bash line-coverage gate.** Bash coverage here is behavioral
  only: `cases.json`/`scenarios.json` + `test-lib.sh` direct assertions + the
  Python suites. A line-coverage tool for the shell surface (e.g. `kcov`) was
  assessed and judged over-engineering for this profile — the behavioral
  suite already pins the enforcement logic that matters, and a line-coverage
  number would not improve the signal. This is a deliberate scope decision,
  not an open item.
